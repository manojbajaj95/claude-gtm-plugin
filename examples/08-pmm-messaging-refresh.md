# The PMM Messaging Refresh

Audit your features page against two competitors and rewrite the 5 most technical, benefit-weak descriptions into clear, outcome-driven copy aligned with your GTM strategy.

## Workflow

### Step 1 — Audit the current features page

```
Read the current features page of our website (URL: [Features Page URL]).
For each feature listed, score the description on two axes:
  1. Technical vs. benefit language (1 = pure tech, 5 = pure benefit)
  2. Clarity for a non-technical buyer (1 = confusing, 5 = crystal clear)
Return the 5 lowest-scoring feature descriptions with their current copy.
```

### Step 2 — Competitor comparison

```
Run a competitor analysis on the features pages of [Competitor 1] and [Competitor 2].
For each of our 5 low-scoring features, find how the competitors describe the equivalent feature.
Identify:
  - Where they use more benefit-driven language than we do
  - Where our technical copy might actually confuse buyers who are comparing options
Return a side-by-side comparison for each of the 5 features.
```

### Step 3 — Rewrite the messaging

```
Read strategy/brand.md for our current GTM strategy, positioning, and ICP.
Using the audit from Step 1 and competitor gaps from Step 2, rewrite the copy for
all 5 features.
For each feature deliver:
  - Feature name
  - BEFORE: current copy (verbatim)
  - AFTER: new copy — benefit-first, ICP-specific, under 30 words
  - One-line rationale explaining the change
Format as a messaging document ready to hand off to a web team.
```

## Output

- `strategy/brand.md` — append the revised feature messaging under a "Features Page Messaging" section
- `content/briefs/drafts/YYYY-MM-DD_messaging-refresh.md` — before/after document for the web team
