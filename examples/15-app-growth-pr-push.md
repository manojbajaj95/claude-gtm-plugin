# The App Growth & PR Push

Audit your app store listing, rewrite the title and description with high-volume keywords, then announce the refresh with a press release pitched to industry tech blogs.

## Workflow

### Step 1 — Audit the current app store listing

```
Audit our current app store listing using app store optimization best practices.
App Store URL or app name: [Insert App Name / URL]
Platform: [App Store / Google Play / both]

Evaluate:
  - App title: keyword density, character usage, clarity
  - Subtitle / short description: benefit clarity, keyword inclusion
  - Primary description (first 3 lines — visible before "more"): hook strength,
    primary keyword placement, CTA presence
  - Screenshots and preview: note any copy or text visible in the assets
  - Keyword field (if accessible): gaps or wasted terms

Return: a prioritized list of missing high-volume keywords and the top 3 copy weaknesses.
```

### Step 2 — Research and cluster keywords

```
Conduct keyword research for the app's core use case and target audience.
Cluster the keywords into:
  - Primary keyword (1): highest volume + most relevant, for the title
  - Secondary keywords (3–5): for the subtitle and first paragraph of the description
  - Long-tail keywords (5–8): for mid and lower description, and keyword field

For each keyword include: estimated monthly search volume, competition level, and
whether it matches how users search in the app store vs. Google search.
```

### Step 3 — Rewrite the listing

```
Using the audit from Step 1 and keywords from Step 2, rewrite:
  1. App title: under 30 characters, primary keyword included naturally
  2. Subtitle / short description: under 30 characters, lead with the user benefit
  3. Primary description:
       - First 3 lines (above the fold): hook + primary value prop + primary keyword
       - Full description (up to 4,000 characters): benefit-driven, keyword-rich,
         structured with short paragraphs and line breaks for mobile readability
       - End with a CTA (rate, share, or upgrade prompt)
Deliver clean copy ready to paste into the app store dashboard.
```

### Step 4 — Draft the press release

```
Write a press release announcing the app's optimization update.
Target: industry tech blogs and product/marketing newsletters

Structure:
  - Headline: newsworthy, under 100 characters
  - Dateline and opening paragraph: who, what, when, where, why — in 2 sentences
  - Body (2–3 paragraphs): what changed, why it matters for users, one data point or
    user outcome stat if available
  - Founder or product quote: 1–2 sentences in first person
  - Boilerplate: 3-sentence company description
  - Contact: [Name] | [Email] placeholder

Tone: professional but not corporate — approachable for indie tech and startup media.
Under 400 words total.
```

## Output

- `content/app-store/drafts/YYYY-MM-DD_listing-rewrite-<app-slug>.md` — full rewritten listing
- `content/pr/drafts/YYYY-MM-DD_press-release-<app-slug>.md` — press release
